# 第2组 2026年4月14日 讨论记录

> **项目名称：** 基于 PaddleOCR 与 YOLO 的双栏 PDF 正文提取系统

---

## 一、项目目标

利用 **DocLayout-YOLO**（基于 YOLOv10 优化的文档版面分析模型）自动识别 PDF 每一页中的版面元素（标题、正文、插图、插图附文），并以 **"村落"** 为粒度将插图和对应文本进行关联归档，最终输出结构化的各村图文数据。

### 1.1 核心功能模块

| 模块 | 功能描述 |
|------|----------|
| **版面元素检测** | 识别页面中的 Title（村名标题）、Text（正文段落）、Figure（插图）、Caption（插图附文） |
| **村落边界划分** | 以"村名标题"为锚点，划分每个村落在页面上的内容区域范围 |
| **图文关联匹配** | 将每个村落区域内的插图、附文、正文进行绑定 |
| **结构化输出** | 按 `各村OCR结果/村名/` 目录结构输出插图（以附文命名）和文本 txt 文件 |

### 1.2 预期输出结果格式

```
各村OCR结果/
├── A村/                        # 存放A村对应的所有插图和A村文字描述txt文件
│   ├── A村插图1.jpg            # 从PDF页面中裁剪出的插图，文件名使用插图下方的附文描述
│   ├── A村插图2.jpg
│   └── A村.txt                 # 该村落的正文描述，保持原文段落顺序
├── B村/
│   ├── B村插图1.jpg
│   ├── B村插图2.jpg
│   └── B村.txt
│   ...
└── 全书文本.txt                # 提取出的全书所有文字（这一步功能已经实现）
```

### 1.3 输出示例

```

├── ...

```

## 三、遇到的问题与解决方案

### 3.1 PaddleOCR (CPU版本)环境配置问题

#### ❌ 报错信息

```
NotImplementedError: (Unimplemented) ConvertPirAttribute2RuntimeAttribute not support
[pir::ArrayAttribute<pir::DoubleAttribute>]
```

#### 🔍 问题原因

| 原因 | 说明 |
|------|------|
| 版本兼容性 | PaddlePaddle 3.3.0 版本在 CPU 推理时使用了 oneDNN v3.6.2 |
| 转换层缺陷 | 在将 PIR 属性转换为 Runtime 属性时，`pir::ArrayAttribute<pir::DoubleAttribute>` 类型未被实现支持 |
| 回归缺陷 | 这是 PaddlePaddle 3.3.0 的回归缺陷，在 3.2.x 及更早版本中工作正常 |

#### ✅ 解决方案

**方案一：降级 PaddlePaddle 到 3.2.2（cpu版本推荐）**

```bash
conda create -n paddleocr python=3.9 -y
conda activate paddleocr
python -m pip install --upgrade pip
python -m pip install paddleocr
pip install paddlepaddle==3.2.2   # 卸载 paddlepaddle 的原来 3.3.0 版本（有bug）
```

**方案二：使用 GPU 版本（如果有显卡）**

> 如有 NVIDIA 显卡，可安装 `paddlepaddle-gpu` 版本以规避该 CPU 推理 Bug。

---

### 3.2 正文缩进问题

#### 🔍 问题分析

经过观察，PaddleOCR 默认 JSON 输出结果中带有 `rec_boxes` 参数，该参数记录了识别出的文本的坐标。每个段落的开头（缩进）位置明显与其他行有巨大差异。

#### ✅ 解决方案

利用这一差异，在从 JSON 输出结果里提取 TXT 文本的程序中，设定一个固定阈值 `indent_threshold` 来判断是否缩进，从而正确还原段落结构。

## 四、下一步技术路线安排

```
┌─────────────────────────────────────────────────────────────────┐
│                     输入：PDF 文件                               │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  Stage 1: PDF → 页面图像转换                                      │
│  工具：pdf2image / PyMuPDF                                        │
│  输出：每页 PNG/JPG 图像（DPI ≥ 300，保证 OCR 质量）               │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  Stage 2: 版面分析（DocLayout-YOLO）                              │
│  模型：DocLayout-YOLO（基于 YOLOv10，预训练于 DocSynth-300K）     │
│  检测类别：Title, Text, Figure, Caption, Table, Header, Footer   │
│  输出：每页元素的 bounding box + 类别 + 置信度                    │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  Stage 3: 版面元素后处理与村落区域划分                             │
│  逻辑：以 Title（村名）为锚点，按 Y 轴位置划分纵向内容区块          │
│        每个区块内关联 Figure + Caption + Text                     │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  Stage 4: OCR 文字提取（已部分实现）                               │
│  工具：PaddleOCR / Tesseract / 已有 OCR 模块                      │
│  输入：Text 区域图像 → 输出：纯文本                                │
│  输入：Caption 区域图像 → 输出：附文（用作插图文件名）              │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  Stage 5: 结构化输出                                              │
│  按村落创建目录，保存插图（以附文命名）和 txt 文件                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4.1 各阶段详细说明

| 阶段 | 名称 | 工具/技术 | 输入 | 输出 |
|------|------|-----------|------|------|
| **Stage1** | PDF转图像 | `pdf2image` / `PyMuPDF` | PDF 文件 | 每页 PNG/JPG（DPI ≥ 300） |
| **Stage2** | 版面分析 | `DocLayout-YOLO`（YOLOv10） | 页面图像 | bbox + 类别 + 置信度 |
| **Stage3** | 区域划分 | 自定义后处理算法 | 版面元素 | 村落内容区块 |
| **Stage4** | OCR提取 | `PaddleOCR` | 区域图像 | 纯文本 / 附文 |
| **Stage5** | 结构化输出 | Python 文件操作 | 文本 + 插图 | `各村OCR结果/` 目录结构 |
